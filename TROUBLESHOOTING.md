# 문제 해결 지도 (오류 나면 여기부터)

바이버스는 **역할별로 파일이 나뉘어** 있어서, 문제가 나면 "어디를 봐야 하는지"가 정해져 있어요.
아래 표에서 증상을 찾아 해당 파일만 고치면 됩니다. **고치기 전에 이 파일을 개발자에게 보여주면 훨씬 빨라요.**

## 🗺️ 어디에 뭐가 있나

| 파일 | 담당 | 여기서 고치는 것 |
|---|---|---|
| `src/lib/supabase.js` | 서버 **연결** | 키/URL, mock 폴백 |
| `src/lib/db.js` | **데이터** 읽고 쓰기 | 작품·댓글·좋아요·라이선스·통계·사용자 쿼리 |
| `src/App.jsx` | **화면**과 흐름 | 버튼 동작, 페이지 전환, 폼 |
| `src/ErrorBoundary.jsx` | **안전망** | 앱이 멈췄을 때 뜨는 안내 화면 |
| `supabase/*.sql` | **DB 설계** | 테이블·권한(RLS)·트리거 |
| `supabase/functions/delete-user/` | 사용자 삭제 **서버 함수** | 계정 삭제 로직 |

## 🔧 증상 → 원인 → 해결

| 화면에 뜬 증상 | 원인 | 어디를 고치나 |
|---|---|---|
| 🛠️ "화면에 문제가 생겼어요" | 코드 오류로 화면이 멈춤 | 회색 상자(오류 내용)를 복사해 개발자에게 → `App.jsx` 대개 |
| 노란 "미리보기 모드" 배너 | Supabase 키 누락 | 로컬은 `.env.local`, 배포는 호스팅 환경변수 |
| "작품을 불러오지 못했습니다" | 테이블 미생성 | `schema.sql` + `schema_marketplace.sql` 실행 |
| "댓글 등록에 실패했습니다" | 로그인 안 됨 / RLS | 로그인 상태 확인 → `db.js addComment` |
| "삭제에 실패했습니다 (권한 확인)" | 관리자 아님 / RLS | `ADMIN_SETUP.md` STEP 1 (role='admin') |
| 관리자 콘솔 "누적 매출"이 0 | 매출 정책 미적용 | `schema_admin.sql` 실행 |
| 사용자 "삭제" 눌러도 안 지워짐 | 삭제 함수 미배포 | `ADMIN_SETUP.md` STEP 3 |
| 로그인이 새로고침 때 풀림 | 배포 주소 미등록 | Supabase → Auth → URL Configuration |
| 가입 확인 메일 안 옴 | 무료 메일 한도(2통/시간) | 잠시 후 재시도 / 실서비스는 SMTP |
| 빌드 실패(Build Failed) | 보통 문법/설정 오류 | 터미널의 빨간 메시지 복사해 문의 |

## 🧪 문제가 진짜 서버 문제인지 1분 만에 확인

터미널에서 `vibers` 폴더 안에 두고 실행하면, 서버가 데이터를 제대로 막고 있는지 바로 보여요:
```bash
npm run build   # 코드가 깨졌는지: 통과하면 코드 문제 아님
```
브라우저 개발자도구(F12) → Console 탭에 **빨간 글씨**가 있으면, 그 내용이 원인 힌트예요. 복사해서 주세요.

## 💾 되돌리기 (git)

커밋을 해두면 언제든 마지막 정상 상태로 되돌릴 수 있어요:
```bash
git status              # 뭐가 바뀌었는지
git restore <파일>       # 특정 파일을 마지막 커밋 상태로 되돌리기
git stash               # 바꾼 것 전부 임시로 치우기 (git stash pop 으로 복구)
```
> 큰 변경 전에는 `git add -A && git commit -m "동작 확인된 상태"` 로 한 번 저장해두면, 뭘 해도 안전하게 돌아올 수 있어요.
